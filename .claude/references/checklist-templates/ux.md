# 🎨 UX Checklist Template

> Domain: User Experience Validation
> Used by: `/checklist-generator` for UX audits

## Interaction Design

- [ ] [CL-UX-001] Primary action (CTA) is visually prominent on every screen
- [ ] [CL-UX-002] Destructive actions require confirmation (delete, cancel, discard)
- [ ] [CL-UX-003] Form validation provides inline, real-time feedback
- [ ] [CL-UX-004] Navigation is consistent across all pages

## States & Feedback

- [ ] [CL-UX-005] Loading states shown for all async operations
- [ ] [CL-UX-006] Empty states provide guidance (not blank screens)
- [ ] [CL-UX-007] Error states include specific message + recovery action
- [ ] [CL-UX-008] Success confirmations are visible and non-blocking

## Accessibility

- [ ] [CL-UX-009] All interactive elements are keyboard navigable
- [ ] [CL-UX-010] Color is not the sole conveyor of information
- [ ] [CL-UX-011] Touch targets are ≥ 44x44 CSS pixels
- [ ] [CL-UX-012] Form labels are associated with inputs (not placeholder-only)
- [ ] [CL-UX-013] Alt text provided for all informational images

## Responsiveness

- [ ] [CL-UX-014] Layout adapts to mobile, tablet, and desktop viewports
- [ ] [CL-UX-015] No horizontal scrolling on mobile
- [ ] [CL-UX-016] Text remains readable without zooming on all devices

## Content

- [ ] [CL-UX-017] Error messages are human-readable (no error codes without explanation)
- [ ] [CL-UX-018] Microcopy is consistent (same term for same action everywhere)
- [ ] [CL-UX-019] Date/time formats respect user locale expectations
- [ ] [CL-UX-020] Pagination or infinite scroll for lists > 20 items
