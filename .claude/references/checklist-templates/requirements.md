# ✅ Requirements Checklist Template

> Domain: Specification Quality Validation
> Used by: `/checklist-generator` for requirements audits

## Completeness

- [ ] [CL-REQ-001] Every user story has at least one GIVEN/WHEN/THEN acceptance criterion
- [ ] [CL-REQ-002] Every AC has a measurable outcome (no vague adjectives)
- [ ] [CL-REQ-003] All key entities are defined with core attributes and relationships
- [ ] [CL-REQ-004] Explicit non-goals section exists and is specific
- [ ] [CL-REQ-005] Constraining assumptions are documented

## Clarity

- [ ] [CL-REQ-006] No unresolved `[NEEDS CLARIFICATION]` markers remain
- [ ] [CL-REQ-007] No placeholder text (`TBD`, `TODO`, `<placeholder>`)
- [ ] [CL-REQ-008] Terminology is consistent throughout (no synonyms for same concept)
- [ ] [CL-REQ-009] Each AC can produce a deterministic test (pass or fail, no "maybe")

## Priority & Scope

- [ ] [CL-REQ-010] User stories are prioritized (P1, P2, P3)
- [ ] [CL-REQ-011] Each story has an independent testability statement
- [ ] [CL-REQ-012] MVP scope is clearly identified

## Traceability

- [ ] [CL-REQ-013] Every requirement traces to a business goal
- [ ] [CL-REQ-014] No duplicate or near-duplicate requirements
- [ ] [CL-REQ-015] Security and performance requirements are explicitly stated (not assumed)
